# Contributing to Port Scanner

Thanks for your interest in contributing! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/port-scanner.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Guidelines

### Code Style

- Follow PEP 8 Python style guidelines
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and single-purpose

### Testing

- Test your changes on multiple targets (with permission)
- Verify edge cases (invalid IPs, unreachable hosts, etc.)
- Make sure existing functionality still works

### Commit Messages

- Use clear, descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Keep the first line under 50 characters
- Add details in the body if needed

### Pull Requests

- Describe what your PR does and why
- Reference any related issues
- Make sure your code follows the style guidelines
- Update documentation if needed

## Ideas for Contributions

- Add UDP port scanning
- Implement OS fingerprinting
- Add output formats (JSON, CSV, XML)
- Improve performance optimizations
- Add more comprehensive error handling
- Create automated tests
- Improve documentation
- Add banner grabbing
- Implement stealth scanning techniques

## Questions?

Open an issue for questions or discussions about potential contributions.

## Legal Notice

Remember: only scan systems you own or have explicit permission to test. Contributors must ensure their code respects this principle.
